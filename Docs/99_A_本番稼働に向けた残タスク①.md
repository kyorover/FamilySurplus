## 📝 ユーザー側残作業（TODO）

### 1. 法的文書のプレースホルダー更新
Notion等で公開した「利用規約」および「プライバシーポリシー」の本文を修正してください。
- **対象項目**: 文中の `[ ]` で囲まれた以下の情報を実情に合わせて書き換え。
    - 運営者名（または屋号）
    - 所在地（公開可能な範囲）
    - 連絡先メールアドレス
    - 制定日（202X年X月X日）

### 2. 【リリース必須】AdMobの本番App ID取得と設定
AdMob管理画面から本番用IDを取得し、コードに反映してください。
- **対象ファイル**: `app.config.ts`
- **書き換え箇所**: 
    - `ADMOB_TEST_APP_ID_ANDROID` → あなたの「AndroidアプリID」
    - `ADMOB_TEST_APP_ID_IOS` → あなたの「iOSアプリID」
- **対象ファイル**: `src/components/dashboard/DashboardAdBanner.tsx`
    - `UNIT_ID_IOS` / `UNIT_ID_ANDROID` → あなたの「バナー広告ユニットID」

### 3. 【CI/CD稼働必須】AWS OIDC設定とGitHub Secretsの登録
GitHub ActionsからAWSへ安全にデプロイするため、AWS側でOIDCプロバイダとIAMロールを作成し、GitHubに登録してください。
- **設定場所**: AWS IAM (IDプロバイダ、ロール) および GitHubリポジトリの Settings > Secrets and variables > Actions
- **設定内容**:
    - `AWS_ROLE_ARN_PROD` という名前のSecretを作成し、本番用デプロイロールのARNを登録。
    - `AWS_ROLE_ARN_DEV` という名前のSecretを作成し、開発用デプロイロールのARNを登録。

### 4. 【通信必須】API GatewayエンドポイントURLの反映
AWSへCDKをデプロイした際に出力されるAPIのURLを、アプリに設定してください。
- **対象ファイル**: `app.config.ts`
- **書き換え箇所**: `API_URLS` オブジェクト内のプレースホルダーを実際のURLに書き換え。
    - `development: 'https://your-dev-api-id.execute-api.ap-northeast-1.amazonaws.com/prod/'` → 開発用スタック(`hesokuri-backend-dev`)のApiEndpointUrl
    - `production: 'https://your-prod-api-id.execute-api.ap-northeast-1.amazonaws.com/prod/'` → 本番用スタック(`hesokuri-backend-prod`)のApiEndpointUrl
