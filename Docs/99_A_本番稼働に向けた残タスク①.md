# 【残タスク】App Store公開要件整理とタスク定義

| 大項目 | タスク名 | 具体的な仕様・制約（推測排除） | 影響する既存ファイル / 新規追加要素 |
| :--- | :--- | :--- | :--- |
| **1. Apple審査必須対応** | 1-1. アカウント削除（退会）機能 | ユーザー自身による即時退会とデータ物理削除の導線が必須（ガイドライン5.1.1(v)）。 | UI: `SettingsScreen.tsx`<br>API: `apiService.ts` に削除リクエスト追加<br>AWS: `lambda/index.ts` にDELETE API追加、Cognitoユーザー及びDynamoDB該当データ削除 |
| | 1-2. プライバシーポリシー/利用規約 | 登録時の同意チェックボックス必須。設定画面からのリンク参照必須。 | UI: `AuthInputForm.tsx` (同意UI追加)<br>UI: `SettingsScreen.tsx` (リンク追加) |
| | 1-3. ATT（トラッキング許諾）対応 | 広告枠切り替えを実装するため、iOS14以降必須のATT許諾ダイアログを表示する。 | 設定: `app.json` (権限記述追加)<br>UI: 初期化ロジック (許諾ダイアログの呼び出し) |
| **2. 課金・プラン・広告管理** | 2-1. スキーマ拡張 | `AccountInfo` に課金プランと有効期限を追加。※収入(income等)の概念・入力は絶対に追加しない。 | 型: `src/types.ts` (`subscriptionPlan` の拡張など) |
| | 2-2. DBカラム追加 (DynamoDB) | 既存のアカウントテーブルに対し、上記属性を追加保存する。 | AWS: `AccountsTable` (既存のPKに対して属性追加のみ。GSI追加は不要) |
| | 2-3. ダッシュボード広告実装 | ユーザーのアカウント情報（`subscriptionPlan === 'FREE'`）を判定し、無課金状態の場合はダッシュボード上にバナー広告（AdMob等）を常時表示する。 | 設定: `app.json` (SDK設定)<br>UI: `DashboardScreen.tsx` (広告コンポーネント配置) |
| **3. インフラ・CI/CD構築** | 3-1. 環境分離（Dev/Prod）の構成 | CDKを用いて開発環境（Dev）と本番環境（Prod）のスタックを分離。API Gateway URLやDynamoDBテーブルを物理的に分け、テストデータが本番に混入するのを防ぐ。 | AWS: `bin/hesokuri-backend.ts`<br>AWS: `lib/hesokuri-backend-stack.ts` |
| | 3-2. 自動デプロイ（CI/CD）の構築 | GitHub Actions等を導入し、特定のブランチ（main等）へのマージをトリガーとして、自動で `npx cdk deploy` がAWS本番環境へ実行されるパイプラインを構築する。 | 新規: `.github/workflows/deploy.yml` |
