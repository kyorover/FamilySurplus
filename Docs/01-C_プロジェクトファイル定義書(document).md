# 01-C_プロジェクトファイル定義書(document)

## 1. プロジェクト・ドキュメント群 (Docs/, memo/)
プロジェクトの仕様・ルールを規定する文書。コード生成時はこれらに反する実装を行わないこと。

| ファイルパス | AI向け責務・制約（コーディング時の前提知識） |
| :--- | :--- |
| Docs/00-A_へそくり帳_企画書.md | プロジェクトの目的、ターゲット、基本コンセプト。UI/UXの方向性の根拠。 |
| Docs/00-B_へそくり帳_憲法.md | 【最重要】開発における絶対遵守ルール。**「収入（income等）の完全排除」「固定科目の扱い（削除・名称変更不可）」**などが定義される。コード生成前に必ず意識すべき制約。 |
| Docs/01-A_へそくり帳_要件定義書.md | アプリケーションが備えるべき機能要件のリスト。不足機能の補完や仕様理解に用いる。 |
| Docs/01-B_へそくり帳_方式設計書.md| AWSアーキテクチャ（サーバーレス）やフロントエンド（React Native/Expo）の技術スタック・通信方式の定義。 |
| Docs/02-D_DynamoDB仕様書.md | DynamoDBのテーブル設計、キー設計（PK/SK）、およびGSIの仕様を定義する文書。 |
