プレミアムプラン（サブスクリプション）検証手順書 (iOS専用)

本ドキュメントは、アプリ内課金（サブスクリプション）の実装が正常に動作し、バックエンド（DynamoDB）との同期およびアプリ内の権限解放（広告非表示等）が正しく行われるかを検証するための手順書です。

【重要：Windows環境からiPhone実機へのデプロイについて】
アプリ内課金（Appleのネイティブ決済機能）は「Expo Go」アプリでは動作しません（エラーになります）。
Windows環境で開発し、iPhone実機で検証を行うには、Expoのクラウドビルド（EAS Build）を使用して、Appleの「TestFlight」経由でインストールするのが最も確実な手法です。


【Apple Developer Program 登録・支払い手順】

1. Apple IDの準備
2ファクタ認証が有効になっているApple IDを用意します。

2. Apple Developer アプリのインストール
iPhoneのApp Storeから「Apple Developer」アプリをインストールします。（※Webブラウザからも登録可能ですが、免許証等の本人確認プロセスがあるため、iPhoneアプリ経由で行うのが最も確実で推奨されています）

3. 登録（Enroll）の開始
・アプリを開き「アカウント」タブから該当のApple IDでサインインします。
・「今すぐ登録（Enroll Now）」をタップします。

4. 個人 / 法人情報の入力
・個人として登録するか、法人（マイクロ法人など）として登録するかを選択します。
・法人の場合は「D-U-N-S番号（企業識別番号）」の入力が求められます（未取得の場合はAppleのサイト経由で無料申請可能）。
・画面の指示に従い、氏名、住所、電話番号などを入力し、身分証による本人確認プロセスを完了させます。

5. 年会費の支払い（Appleへの課金）
・入力完了後、サブスクリプションの購入画面が表示されます。
・iPhoneに紐づいている支払い方法（クレジットカード等）で、年間約14,800円の決済を完了させます。

6. 承認の完了
・決済後、Apple側でアカウントの審査が行われます（通常数時間〜48時間程度）。
・完了メールが届けば登録完了です。これにより、EAS Buildでの証明書発行、App Store Connectへのアクセス、およびTestFlightでの検証がすべて解放されます。

---
1. ビルド手順と実機への導入方法

[1-1. EAS CLIの準備とログイン]
以下のコマンドを順に実行してください：

  npm install -g eas-cli
  eas login
  eas build:configure

[1-2. テスト用ビルドの作成（クラウドビルド）]
Windows端末のターミナルで以下のコマンドを実行し、Appleのサーバーへビルドとアップロードを行います。

  eas build --platform ios --profile production
  eas submit --platform ios

※アップロード完了後、App Store Connect側で輸出コンプライアンス等を設定し、iPhoneアプリの「TestFlight」から実機にインストールしてください。

---
2. 課金テスト環境（Sandbox）の準備

実際のクレジットカードで決済されないよう、Appleが提供するテストアカウントを使用します。

1. PCブラウザで App Store Connect (https://appstoreconnect.apple.com/) にログイン。
2. 「ユーザーとアクセス」 > 左メニュー「Sandboxテスター」を選択し、「＋」からテスト用のApple IDを新規作成。
3. iPhone実機の操作: 「設定」アプリ > 「App Store」 > 画面最下部の「SANDBOXアカウント」に、作成したテスト用IDでサインインする。

---
3. テスト実施手順

実機（TestFlight版）でアプリを起動し、以下の手順で課金フローを実行します。

1. アプリ下部タブから「設定」画面を開く。
2. 「💎 プレミアムプラン」セクションの「プランの購入・復元」をタップし、Paywallモーダルを開く。
3. ローディング後、プラン（月額 / 年額）が表示されることを確認。
4. いずれかのプランの購入ボタンをタップする。
5. OS標準の決済シートが立ち上がることを確認する。
   ※この時、必ず「[Environment: Sandbox]」というテスト環境である旨の表記が出ていることを確認してください。
6. パスワード入力やFaceID等を行い、決済を承認する。
7. アプリ画面に「プレミアムプランの購入が完了しました！」のアラートが表示されることを確認し、閉じる。

---
4. 期待値と確認項目（検証完了の定義）

上記の手順を実行後、以下の3点すべてが想定通りになっていることをもって「検証成功」とします。

① DynamoDBのフラグ更新確認（バックエンド同期）
Webhookが正常に動作し、データベースが書き換わっているかを確認します。
1. AWSマネジメントコンソールから DynamoDB > 項目の探索 を開く。
2. 対象環境の AccountsTable を選択。
3. テストしたユーザーの accountId (Cognitoのsub) のレコードを確認する。
【期待値】
・subscriptionPlan が "FREE" から "PREMIUM" に更新されていること。
・subscriptionExpiry に有効期限のタイムスタンプ（ISO 8601形式）が記録されていること。

② アプリ内への権限反映確認（フロントエンド同線）
1. 決済完了後、アプリを一度タスクキル（上スワイプで完全終了）する。
2. アプリを再起動する。（起動時の fetchAccountInfo() によって最新状態が取得される）
【期待値】
・ダッシュボードや入力画面の下部などに配置されている広告バナーが、完全に非表示（レイアウト上からも消失）になっていること。

③ RevenueCat ダッシュボードでの確認
1. RevenueCat (https://app.revenuecat.com/) にログインし、対象プロジェクトを開く。
2. 「Customers」からテストしたユーザーのID（accountId）を検索。
【期待値】
・Active Entitlements に premium が付与されていること。
・画面右側の「Customer History」にて、決済イベントが記録されており、AWSへのWebhook（INITIAL_PURCHASE 等）が HTTP 200 OK で送信成功していること。
