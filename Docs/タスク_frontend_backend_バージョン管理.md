# 【引継ぎ資料】バックエンド自動デプロイの廃止およびアプリバージョン検証機構の導入

## 1. 背景と課題
現在、バックエンド（AWS CDK）のデプロイパイプラインは、`main` または `develop` ブランチへの Push をトリガーとして自動実行される設定（`deploy.yml`）となっている。
しかし、本プロジェクトは React Native / Expo によるモバイルアプリであり、Webアプリとは異なり「バックエンドの更新と全ユーザーのフロントエンドの更新が完全に同期しない」という特性を持つ。
そのため、アプリのストアリリース（バージョンアップ）とは無関係にバックエンドが自動更新されると、アップデートを行っていない既存ユーザーのアプリが新仕様のバックエンドにアクセスし、クラッシュ等の重大な障害を引き起こすリスク（障害の温床）が存在する。

## 2. 目的
1. バックエンドの自動デプロイを廃止し、アプリのリリースサイクルに合わせた任意のタイミング（手動）でのみ本番環境を更新できるようにする。
2. アプリの起動時にバックエンドとの互換性を検証する機構を導入し、バージョン不一致による障害を未然に防ぐ。
3. 互換性のない旧アプリを使用しているユーザーに対し、強制アップデート要求やメンテナンス画面を適切に表示し、安全なUXを提供する。

## 3. システム要件
本対応において、以下の4つの要件を満たすシステム改修を実施する。

1. **自動デプロイの無効化**
   - GitHub への Push による自動発火を停止し、リリース担当者の手動操作（`workflow_dispatch`）によるデプロイへ移行する。
2. **バックエンド・バージョン検証機構の導入**
   - アプリ起動時に呼び出す、認証不要の「アプリステータスAPI（例: `GET /system/status`）」を新設する。
   - レスポンスとして、`minAppVersion`（許容する最低アプリバージョン）、`latestAppVersion`、`isMaintenance`（メンテナンスフラグ）を返却する。
3. **強制アップデートルーチンの組み込み**
   - 端末のアプリバージョンが `minAppVersion` を下回る場合、ストア（App Store / Google Play）への遷移ボタンのみを持つ専用画面を表示し、アプリの利用をブロックする。
4. **メンテナンス・先行リリース時のフェイルセーフ**
   - APIから `isMaintenance: true` が返却された場合、または APIのレスポンス形式がパースできない（バックエンドが古すぎる）場合は、「メンテナンス中」の専用画面を表示し、データの操作をブロックする。

## 4. 修正方針
上記要件を実現するため、インフラ、バックエンド、フロントエンドの3層に分けて以下の改修を行う。

### 4.1. インフラ（CI/CD）
- `deploy.yml` の `on` セクションを `push` から `workflow_dispatch`（手動実行・環境選択式）に変更する。

### 4.2. バックエンド (AWS CDK / Lambda)
- システムの稼働状態や要求バージョンを管理するための設定ストア（DynamoDBテーブル、または環境変数/SSMパラメータ）を新規構築する。
- 認証不要（API GatewayでIAM認証等をかけない）の新規 Lambda 関数を作成し、上記設定を返却するエンドポイント（`/system/status`）を公開する。

### 4.3. フロントエンド (Expo / React Native)
- `App.tsx` またはそれに準ずるアプリの最上位階層（スプラッシュスクリーン表示中等）に、ステータスAPIをフェッチする初期化ロジックを追加する。
- 状態に応じた分岐処理を実装し、通常画面、強制アップデート要求画面、メンテナンス画面を出し分ける。
- ネイティブアプリのバージョン番号（`app.config.ts` 等で定義される version）を実行時に取得する仕組み（`expo-constants` や `expo-application` の利用）を導入する。

## 5. 対象資材（影響範囲）
※以下は想定される主要な変更・追加対象ファイル

**■ インフラ層**
- `hesokuri-backend/.github/workflows/deploy.yml` （変更: トリガーの修正）

**■ バックエンド層 (`hesokuri-backend/`)**
- `lib/hesokuri-backend-stack.ts` （変更: 新規APIおよびDynamoDB等のリソース定義追加）
- `lambda/controllers/systemController.ts` （新規: ステータスAPI用Lambdaハンドラー）
- `lambda/index.ts` （変更: ルーティングへの追加）

**■ フロントエンド層 (`hesokuri-cho/`)**
- `App.tsx` または 初期化処理を担うプロバイダー （変更: 起動時のAPIチェックロジック追加）
- `src/services/apiService.ts` （変更: `/system/status` のフェッチ関数追加）
- `src/screens/ForceUpdateScreen.tsx` （新規: 強制アップデート画面コンポーネント）
- `src/screens/MaintenanceScreen.tsx` （新規: メンテナンス画面コンポーネント）
- `app.config.ts` または `package.json` （確認・変更: バージョン管理情報の整備）
